Задача состоит в реализации регистрации которая состоит из:
  Экрана приветствия
  Первого шага регистрации
  Второго шага регистрации
  Процесса загрузки
  Экрана результата регистрации

Макет проекта:
  https://www.figma.com/file/4VcAgfeipM0JhgpEzcgzqD/Book-App-UI-Module

- Использовать Реакт, разделить код на логические компоненты, каждый в своем файле
- Визуальная часть является менее значимой чем функциональная
- Макет - мобильное приложение, поэтому для десктопа необходимо центрировать контент, а фон растягивать на весь экран
- Главная цель - рабочая регистрация,
  вторая - рабочие проверки и дополнительные фишки
  третья - соответствие дизайну

Стартовый экран
  Показывается сразу при загрузке страницы
  Зеленая кнопка перебрасывает на 1-й шаг регистрации

Первый шаг регистрации
  - Инпут с подписью рекомендуется вынести в отдельный компонент.
  - name должно валидироваться (2 и более не пробельных символа)
  - username должно валидироваться (2 и более не пробельных символа)
  - Если значение инпута не проходит валидацию - подсвечивается красным цветом.
  - Страна и Возраст - необязательные поля
  - При нажатии на кнопку NEXT STEP, необходимо отправить запрос на сервер, проверьте не занят ли введенный username
  - https://authserver.worldthirteen.now.sh/check_username?username=something
  - Метод GET, в ответе JSON-объект с ключом status: “OK” - если свободен, “CONFLICT” - Если username занят то подсветить красным поле username
  - Если username свободен то показать шаг 2
  - Кнопка Login - пустая ссылка

Второй шаг регистрации
  - Инпут мейла должен валидироваться
  - Инпут пароля должен валидироваться - минимум 6 символов, и минимум один из них число
  - При нажатии на иконку “глаза” символы пароля должны стать видимыми, повторное нажатие заново скроет
  - Для возможности нажатия на кнопку пароли в двух инпутах должны совпадать
  - При нажатии на кнопку Sign UP открыть экран LOADING, и сделать запрос на сервер
  https://authserver.worldthirteen.now.sh/register
  Метод POST, в качестве body, JSON-объект:
{ name, username, email, password, country, age }.
В ответе придет успешный или неуспешный ответ, формат ответа необходимо изучить и понять самостоятельно. И в зависимости от результата показать экран успешной регистрации или ошибки.
